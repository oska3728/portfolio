001 : Sending e-mails

Here, I would like to present my program written in Python

The program is used as a remainder or a tool for repeating things that we want to learn

Oskar Unikowski

First of all, let's import the necessary libraries

Now, let's take a look at the code. The important thing is that, if we want to use this code, we need to type the emails (our and person's we want to send it to), in the same line as the python command e.g. python3 email_send.py our@email.com someones@email.com

The next step that I have decided to do in this project is some sort of automation. As you can see the code is trying to detect to which service our email belongs. If it is on Gmail, Hotmail or yahoo. If the program cannot detect it, it will ask us for type it in manually.
If you want to read more about smtp click here

After we inserted our mail, we need to type in our password. Don't worry! The password will not show up.

In the next line of code why decide at what time we want our mail to be sent/to be received.

At this point, our loop starts. As we can see the loop equals True. In Python, this way, we create an infinite loop. The first thing that our loop will do is choosing semi-random number. Why? You will get to know further in this article. After that the program assigns to variable "now" part of datetime library - "now" (we imported the whole library previously). Then, again, the program assigns the value of "now" to "current_time" in the following format: HH/MM/SS.

Now, if the current time that has been assigned above equals to the time that we have set before, we create variable "email" that equivalents to "EmailMessage" that we have imported from "email.message" and the program fills up our email's details. In line 45, we create "random_image", it is our variable for the content of our email. "random-image" is a semi-random text file (from 1 to 7) that should be in the same folder as our python program. In each of these text files, please write some notes or something that you want to receive.

The last part of the code is responsible for the actual connection to our email's distribution.
First, we send "hello" ("ehlo") message to enable more options and to see which extensions are supported by the server. Read more about hello message here
Then we start TLS (click here to learn more about Transport Layer Security)
After it is done, we can safely login to our email and send the message.
You may also ask about "try" and "except" that you already have seen in this code. It is here in case if something went wrong. There are hints what may cause the problem that has occurred
